# Security policy — Hermes One-Click

This policy applies to **this repository** (the Hermes One-Click export: Tauri desktop app, bundled runtime, and related packaging). It does **not** replace upstream Hermes / Nous Research policies for code that originates from those projects; see **`THIRD_PARTY_NOTICES.md`** and upstream repositories for their reporting channels.

## Reporting a vulnerability

Use **[GitHub private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability)** for **this** repo: **Security** tab → **Report a vulnerability**. Do not open public issues for undisclosed security defects.

Include: short title, affected paths or components, reproduction steps, impact, and versions / commit SHA if known.

## Scope notes

- Supply-chain issues (dependencies, installer tooling, embedded Python bootstrap) are in scope.
- Issues that only apply to upstream’s hosted services or their official distribution channels should be directed to the appropriate upstream security contacts when relevant.
